# ATX-SPEC-003 Capability Model

## Status

Draft stub v0.1

## Purpose

This document will define fine-grained authority within Atarix.

## Required Topics

- Capability structure
- Rights masks
- Ring interaction
- Delegation rules
- Revocation semantics
- Capability lifetime
- Capability persistence
- Capability transport
- Capability serialization
- Capability audit

## Required Questions

- Is a capability a token, object, or both?
- Can capabilities cross nodes?
- Can capabilities survive reboot?
- How are capabilities serialized?
- How are capabilities revoked?
- How are delegated capabilities constrained?

## Dependency

This specification depends on ATX-SPEC-001 Security Model and ATX-SPEC-002 Authority Model.
