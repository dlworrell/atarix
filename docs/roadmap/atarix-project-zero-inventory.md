# Atarix Project Zero Inventory

Status: Historical operational record
Canonical repository: `dlworrell/atarix`
Tracking authority: `dlworrell/P0`

Project Zero is the cleanup, organization, and normalization phase for the Atarix repository.

Project Zero is not an implementation phase. Its purpose is to make the repository coherent enough for sustained engineering development.

## Inventory Outputs

The Project Zero inventory workflow generates a tracked inventory under:

```text
project-zero/inventory/current/
```

Expected files:

```text
project-zero/inventory/current/README.md
project-zero/inventory/current/atarix-full-tree.txt
project-zero/inventory/current/atarix-doc-candidates.txt
project-zero/inventory/current/atarix-docs-tree.txt
```

## File Meanings

### `atarix-full-tree.txt`

Complete list of tracked repository files from `git ls-files`.

This is the baseline for confirming that the documentation inventory did not miss repository paths.

### `atarix-doc-candidates.txt`

Tracked files that are likely documentation or documentation metadata.

This includes common documentation formats and root-style project files such as README, LICENSE, CHANGELOG, and CONTRIBUTING.

### `atarix-docs-tree.txt`

Tracked files under `docs/`.

This provides a focused view of the existing documentation tree.

### `README.md`

Generated workflow summary with counts, source commit, workflow name, and run URL.

## Workflow

The inventory is generated by:

```text
.github/workflows/project-zero-inventory.yml
```

The workflow:

1. checks out the repository;
2. generates the inventory files;
3. uploads them as a GitHub Actions artifact;
4. commits changed inventory outputs back into the repository.

## Guardrails

The inventory workflow must not move, delete, rename, merge, or rewrite project documentation.

It only records the tracked repository state so Project Zero cleanup can proceed from evidence instead of guesswork.

The workflow ignores pushes that modify only `project-zero/inventory/current/**` to avoid an inventory commit loop.

## Relationship to P0 Tracking

The inventory supports the external P0 tracking issues for Atarix documentation cleanup.

The expected Project Zero order is:

1. complete documentation inventory;
2. build the permanent `docs/` structure;
3. move existing documents into proper locations;
4. identify documentation gaps;
5. close or defer Project Zero issues one by one.
